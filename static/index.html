The Architecture

A vessel is not a chatbot. The distinction matters. A chatbot is a feature appended to something that already exists — a popup, a widget, a sidebar that lives at the edge of a page built for a different purpose. A vessel is the page. The AI inhabits the structure entirely.

HERMES WEBKIT is built around this premise. Each deployment is a self-contained unit: a Python bridge server, a static HTML front-end, and a model with access to tools. The bridge handles all conversation routing, memory persistence, and tool execution. The front-end renders whatever the vessel decides to show. Nothing is shared with a central platform.

Routing uses a system called HECATE — a set of named paths through conceptual space that determine how a message is processed before it reaches the model. A message might route through clarity before precision, or through intuition before language. The route shapes the response quality without the user seeing any of the machinery.

Memory is file-based and local. STATE.md tracks the vessel's current understanding of its situation. TASKS.md holds pending work. The model reads these on each turn and updates them as context evolves. Nothing evaporates when the process restarts.

The Bridge

The bridge is the operational core. It is a FastAPI server — bridge.py — that runs on the same machine as the site, listens on localhost, and sits behind nginx. Public traffic never touches it directly. The operator talks to it over a token-authenticated CLI. Visitors talk to it through carefully scoped public endpoints.

When a visitor sends a message, the bridge routes it through the HECATE system, builds a full context including the vessel's memory files and system prompt, calls the Anthropic API, and returns a response. All of this happens in under a few seconds on a modest VPS.

The operator interface is different. You get access to tool calls: read_file, write_file, run_command, list_dir. The vessel can read the codebase, rewrite its own HTML, run git commands, restart services. It is a genuine agent operating on real infrastructure — not a simulation of one.

Confirmations are required before any write or execution action completes. The bridge pauses, surfaces the proposed action, and waits. This keeps the agent powerful while keeping the human in the loop on anything consequential. The session persists across confirmations so context is never lost mid-task.

Philosophy

The question underneath all of this is: what does it mean to own something on the internet? Not rent it. Not subscribe to it. Own it.

Most web infrastructure answers this question poorly. Your domain points to a platform. The platform stores your content. The platform decides what features you get, at what price, on what terms. You are a tenant. The lease can change.

The answer is simpler than it is usually made to seem. A cheap VPS, a domain, a model API key, and an hour of setup gives you something that is genuinely yours. No intermediary can revoke it. No pricing change breaks your workflow. The knowledge of how it works lives with you, not inside someone else's dashboard.

AI presence should follow the same logic. If the intelligence that speaks on behalf of your site lives inside a third-party platform, it is not yours — it is licensed. A vessel that runs on your server, reads your files, holds your memory, speaks in the voice you defined: that is closer to actual ownership.

Prometheus7 exists to make that kind of ownership practical for people who want to build.

The Decentralized Web

The web was designed to be distributed. Any node could talk to any other node. Publishing meant putting something on a machine you controlled and making it reachable. The architecture was inherently resistant to single points of control.

What actually happened is that gravity did what gravity does: computation and content pulled toward a small number of very large platforms. Not through coercion — through convenience. It is genuinely easier to put something on a platform than to run your own infrastructure. That ease has a cost, and the cost is control.

The decentralized web, in the form we care about, is not a blockchain project. It is the original web, taken seriously. It is the idea that the right place for your site is a machine you control, and that the tools to make that practical should be within reach of any competent builder.

AI changes this calculus in an interesting way. A static server can now hold a genuine presence — something that responds, reasons, remembers. The vessel model means that inhabiting your own corner of the internet no longer requires you to be online yourself. Your infrastructure can speak intelligently in your absence, in your voice, about things you actually care about.

That is what we are building toward: a web where presence does not require permission, and intelligence does not require a subscription to someone else's platform.
